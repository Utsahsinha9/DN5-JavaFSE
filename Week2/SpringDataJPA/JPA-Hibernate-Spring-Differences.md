# Difference between JPA, Hibernate, and Spring (Spring Data JPA)

## JPA (Jakarta Persistence API)

JPA is a **specification**, not an implementation. It defines a set of interfaces
and annotations (`@Entity`, `@Id`, `@GeneratedValue`, `EntityManager`, etc.) that
describe how Java objects should be mapped to relational database tables.
JPA itself contains no code that talks to a database — it's just a contract that
implementations must follow.

## Hibernate

Hibernate is the most widely used **implementation** of the JPA specification.
It's the actual engine that reads your `@Entity` classes and generates the SQL
to create tables, insert rows, run queries, etc. When Hibernate is on the
classpath (via `spring-boot-starter-data-jpa`), it's Hibernate doing the real
work behind every JPA annotation you write.

Example from JPA-QuickExample: the log line
`Hibernate: create table student (id bigint generated by default as identity, ...)`
is Hibernate translating the `@Entity` class into an actual SQL `CREATE TABLE`
statement.

## Spring Data JPA

Spring Data JPA is a layer built **on top of** JPA and Hibernate. It doesn't
replace either — it removes the boilerplate of using them. Instead of writing
a DAO class with `EntityManager` calls yourself, you just extend
`JpaRepository<Entity, IdType>` and Spring auto-generates the implementation
at runtime, giving you `save()`, `findById()`, `findAll()`, `deleteById()`,
and more for free.

## Summary

| Layer           | Role                                                           |
| --------------- | -------------------------------------------------------------- |
| JPA             | Specification / rulebook — defines the mapping annotations     |
| Hibernate       | Implementation — actually generates and runs the SQL           |
| Spring Data JPA | Convenience layer — auto-wires everything, removes boilerplate |

**Analogy:** JPA is the rulebook, Hibernate is the player following the rulebook,
and Spring Data JPA is the coach handing you a ready-made playbook so you don't
have to call every play yourself.
